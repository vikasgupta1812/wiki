Computer Programming

The first skill you will need in order to analyze data is a way to store, extract, and manipulate it.In the first part of this course, we will study a variety of tools to help you do that. First, we will learn python, an all-purpose programming language that is often used for data manipulation. Then, we will study some algorithms and learn to classify them based on their efficiency. Next we will learn some ways data can be stored and how to extract it. SQL is a querying language that is used to pull out specific rows and columns of data from a large spreadsheet-like database. JSON is an alternative way to store data, using keys and values rather than rows and columns. After data storage, we will learn some command-line tools in Unix, such as filesystem basics, SSH, and SCP. Lastly, we will practice using Git, which organize code over time and helps programmers collaborate on the same project.

A Note on Learning Programming:
 
If you are new to computer programming, don’t worry! While it may seem intimidating at first, programming is very learnable, and with practice, you will only get better. One useful tip to remember is that you are free to look up anything and use outside resources to complete the assignments in this path. Memorizing syntax is generally much less useful than being able to use it practically, and when you use something a lot, you will memorize it naturally. In real life, programmers look up functions they need very frequently. Get used to using Google a lot. Wikipedia often gives great overviews of topics. Finally, remember that very few individual problems in this path are, on their own, necessary to be a good data analyst. If you get stuck on something, move on and try to come back to it later.

SQL and JSON

SQL is the most common language for querying (selecting out) data that is stored in a relational database (tables that consist of rows and columns). You should become comfortable querying, grouping, and merging data using SQL.

JSON is another way that data is often stored. It is still semi-structured, but rather than using columns and rows, JSON data is stored as keys and values. The advantage is that it is more flexible and will be more compact for sparse data. On the other hand, it is a bit tricker to work with than data that is stored in a relational database.

Research Design

Research design is a plan to study a scientific or social science problem. Some common examples of research designs include surveys, interviewing, A/B testing, longitudinal studies, and many more. There are many important things to consider when creating a research design in order to make sure you can safely interpret your results the way you intend. For example, let’s say you want to study whether a certain fertilizer is effective. Before you can conduct and experiment and analyze results, you have to decide what you mean by effective and figure out essential details like how many soil samples to observe. In other example, if you conduct a survey, you have to ensure that the questions you ask are unbiased and will be interpreted by your subjects in a way that is consistent with your intentions.

Machine Learning

Let’s say you have a lot of emails, and you want to know which ones are spam and which ones are not. How would you use the text of the emails to find patterns that computers could use to automatically detect spam? This is a classic problem in machine learning, the “construction and study of systems that can learn from data.”1 One general approach is this: take some example emails, some of which are spam, and split it in half. Count which words are common in both the spam and non-spam emails in the first half of the data, and use that to “train” a model of what words to expect when a new unknown email comes in. Then you can check how good your prediction was on the second “test” set of data that you did not use to train your model, and use statistics to assess your model.

Big Data: Hadoop and MapReduce

The scale of data today is very large, and it is getting larger. To process and analyze very large datasets, special techniques must be used to do some of the computations in parallel across many machines. Tech companies like Facebook use Hadoop to handle data on this scale: Hadoop is used in every Facebook product and in a variety of ways. User actions such as a "like" or a status update are stored in a highly distributed, customized MySQL database, but applications such as Facebook Messaging run on top of HBase, Hadoop's NoSQL database framework. All messages sent on desktop or mobile are persisted to HBase. Additionally, the company uses Hadoop and Hive to generate reports for third-party developers and advertisers who need to track the success of their applications or campaigns.

Capstone Projects

The final project is to analyze a set of data, create graphs, and draw conclusions. Choose a set of data of your choice, and then use either R or Python to analyze it. You can choose a dataset from Datamob or anywhere else you like. You could look at NYC Subway data on which trains were late; US college admissions data; or movie reviews. The only requirement is that it includes at least several thousand observations.

Once you choose a dataset that interests you, look through the variables and choose one or more variables that you want to predict based on three more more other variables. Download the dataset in CSV, JSON, TXT, or XML format. Answer your research question(s) by going through the complete research process:
1. Formulate research question(s)
2. Design and implement a study (or in our case, acquire study data)
3. Examine the data for problems
4. Carry out preliminary and then formal analyses
5. Interpret and present results

Make sure to use good research practices, such as recording all notes and methodology along the way in a research journal. Test your model out on test data. You should make sure to use or discuss all of the following in your project:
1. Research design
2. Independent and dependent variables
3. Qualitative and quantitative variables
4. Regression (any type)
5. Machine learning
6. Cross-validation (any type)
7. Null hypothesis testing
8. Sample size
9. At least three different kinds of graphs

Where possible, showcase skills you learned in the elective(s) as well.

Your finished project should be a write-up of your research process and findings. Put your final project into a public folder in Github (along with your code, of course!).

In preparation for this capstone project, one optional final brief review is available. The last week of this Coursera Data Analysis class goes through a basic data analysis problem, from formulating a question through analyzing data and drawing conclusions. If you like, watch lectures 8.1 through 8.4, which give a starting point for what you should do in your capstone project. Of course, you will also go farther in the capstone project, using some of the other techniques that we have covered in this class.
